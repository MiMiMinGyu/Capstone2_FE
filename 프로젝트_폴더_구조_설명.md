# 📁 프로젝트 폴더 구조 및 기능 설명

## 🔧 Core Files (핵심 파일들)
- **`App.jsx`**: 앱의 최상위 컴포넌트, 라우터 설정
- **`main.jsx`**: React 앱 진입점 및 초기화

## 🌐 API Layer (API 계층)
### `api/clients/`
- HTTP 클라이언트 설정 (axios 인스턴스)
- 에러 처리 및 인터셉터 관리
- 사용자 친화적 에러 메시지 생성

### `api/endpoints/`
- API 엔드포인트별 서비스 함수들
- **`chat.js`**: 텔레그램 메시지 관련 API
  - 메시지 가져오기
  - AI 추천 답변 생성
  - 답변 전송
  - 봇 상태 확인
- **`auth.js`**: 인증 관련 API (향후 구현 예정)

### `api/types/`
- API 관련 타입 정의 (향후 TypeScript 도입 시)

## 📱 Components (컴포넌트들)
### `components/`
- 전역에서 재사용 가능한 공통 컴포넌트들

### `features/`
- 기능별로 그룹화된 컴포넌트들

#### `features/chat/components/`
채팅 기능 전용 컴포넌트들:

- **`ChatContainer`**: 채팅 메인 레이아웃
  - 메시지 목록과 AI 추천 패널 통합
  - 모바일/데스크톱 반응형 레이아웃
  - 헤더 및 상태 표시

- **`MessageList`**: 메시지 목록 표시
  - 자동 스크롤 관리
  - 새 메시지 알림
  - 로딩/에러 상태 처리

- **`MessageBubble`**: 개별 메시지 버블
  - 텔레그램 스타일 디자인
  - 읽음/답장 상태 표시
  - AI 추천 버튼

- **`RecommendationPanel`**: AI 추천 답변 패널
  - AI 추천 답변 생성 및 표시
  - 답변 선택 및 전송
  - 로딩 상태 관리

## 🔗 Navigation (내비게이션)
### `router/`
- **`index.jsx`**: 메인 라우팅 설정
- **`ProtectedRoute.jsx`**: 보호된 라우트 컴포넌트 (인증 필요 시)

### `pages/`
- 페이지 레벨 컴포넌트들
- **`pages/chat/`**: 채팅 페이지
  - 서버 메시지와 로컬 메시지 통합 관리
  - SSE(Server-Sent Events) 실시간 메시지 수신
  - 상태 관리 및 에러 처리

## ⚡ Logic & State (로직 및 상태)
### `hooks/`
- 커스텀 훅들 (로직 재사용)

### `stores/`
- 전역 상태 관리 (Zustand/Redux 등)

## 🎨 Styling & Assets (스타일 및 에셋)
### `styles/`
- 전역 스타일, 테마, CSS 변수
- 텔레그램 테마 스타일

### `types/`
- TypeScript 타입 정의들

### `utils/`
- 유틸리티 함수들 (헬퍼, 상수 등)

## 🏗️ 현업 스타일 특징

### 1. Feature-based 구조
- 기능별로 관련 파일들을 그룹화
- 채팅 기능의 모든 컴포넌트가 `features/chat/` 하위에 위치

### 2. Layer 분리
- **API Layer**: 서버 통신 로직
- **Component Layer**: UI 컴포넌트
- **Page Layer**: 비즈니스 로직 및 상태 관리

### 3. 확장성
- 새로운 기능 추가 시 구조적으로 확장 가능
- 각 컴포넌트가 독립적으로 관리됨

### 4. 재사용성
- 컴포넌트와 로직의 재사용성 극대화
- 공통 컴포넌트와 기능별 컴포넌트 분리

## 🔄 리팩토링 완료 현황

### ✅ 완료된 작업
1. **API 폴더**: 디버깅 로그 제거, 한글 주석 추가
2. **Router 구조**: 라우팅 로직 중앙화
3. **Chat 컴포넌트들**: 모든 컴포넌트 한글화 및 최적화
   - ChatContainer: 디버깅 로그 제거, 주석 한글화
   - MessageList: 이미 최적화 완료
   - MessageBubble: 주석 한글화
   - RecommendationPanel: 에러 메시지 한글화

### 🎯 코드 품질 개선
- 에러 검출용 로그는 유지, 디버깅용 로그 제거
- 모든 주석을 한국어로 통일
- 코드 가독성 및 유지보수성 향상
- 현업 표준에 맞는 폴더 구조 적용
